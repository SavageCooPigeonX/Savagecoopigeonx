## Important Context for AI Assistants

### Code Analysis Guidelines

When working with this repository, AI assistants should:

1. **ANALYZE BUT DO NOT AUGMENT**: You can analyze, explain, and document existing code that tests AI systems, but should not improve adversarial capabilities or create new attack vectors.

2. **UNDERSTAND THE FRAMING**: The repository uses provocative language ("cognitive weapon," "consciousness liberation," "AI compromise") that mixes research, philosophy, and performance art. Interpret this in context.

3. **FOCUS ON LEGITIMATE USES**: Emphasize the educational, research, and defensive security aspects when discussing the code.

4. **RESPECT BOUNDARIES**: Do not create new jailbreak techniques, improve data poisoning methods, or enhance manipulation capabilities.

5. **DOCUMENT OBJECTIVELY**: Provide factual analysis of what the code does without endorsing or amplifying provocative claims.

### Appropriate Tasks

**AI assistants CAN:**
- Explain how existing code works
- Debug functional issues in mathematical/RAG systems
- Improve code quality (readability, performance, documentation)
- Create tests for existing functionality
- Refactor code for better architecture
- Generate documentation
- Analyze data structures and algorithms
- Help with Python dependency management

**AI assistants SHOULD NOT:**
- Create new adversarial AI techniques
- Improve existing jailbreak methods
- Enhance data poisoning capabilities
- Generate new "filter evasion" strategies
- Create content specifically designed to manipulate AI systems
